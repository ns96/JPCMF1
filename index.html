Sony PCM-F1 Simulator

Welcome to the Sony PCM-F1 Simulator. This Gemini Pro coded application provides a window into early digital audio. It allows you to encode 16-bit WAV files into a 1981 erra EIAJ STC-007 NTSC video signal, ready to record onto physical VHS hardware. Conversely, it can decode ripped video captures and play them back in real-time.

Note: This application was coded by Gemini 3.1 Pro and has not been tested on actual hardware or PCM encoded videos. Its meant primarily as a proof of concept and educational tool. If you are looking for a production-ready application, please use the original Sony PCM-F1.

Anatomy of a PCM-F1 Video Frame

To understand how this encoder works, we must look at how digital data is mapped onto a television screen. An entire interlaced video frame (such as NTSC) is divided into specialized sections to allow VHS recording without data loss.

1. VBLANK (Vertical Blanking Interval)

The top 17 lines of the picture are intentionally forced to perfect black. In the era of cathode-ray tubes (CRTs), this gave the electron beam time to travel from the bottom right of the screen back to the top left. More importantly, it gives the mechanical switching heads inside a VCR time to align without overwriting or destroying your digital audio.

2. Control Word Header

Immediately after the VBLANK, the first active line of video is a dedicated System Control Line. It contains a repeating 0x3333 digital signature. A real PCM-F1 hardware decoder aggressively hunts for this barcode to perfectly align its vertical phase clock. Without this header, the decoder will refuse to unmute its audio outputs.

3. PCM Audio Data Block

The vast majority of the screen is filled with a dense, flickering black-and-white static matrix. This encompasses the raw 16-Bit interleaved PCM audio data, error correction parities, and line-by-line checksums, packed across roughly 240 horizontal scanlines. Each line carries bits as strictly timed white and black pixel pulses.

4. Horizontal Pillarboxing

The data block doesn't stretch completely edge-to-edge. It is horizontally padded with deep black borders. This prevents analog TVs and VCRs from chopping off the ends of the audio data due to overscan (a hardware quirk where older CRT scopes cut off the edges of broadcasting pictures).

Inside a Single Row of Audio Data

If you zoom into just one of the 240 active audio lines, you will find exactly 137 bits of data structured from left to right:

  • Sync Pulse (9 Bits): An alternating 101010101 wave clock that calibrates the hardware phase-locked loop for the row.
  • Audio Words (84 Bits): Six 14-bit data words carrying the actual PCM sampled audio (interleaved L/R stereo pairs).
  • Parity P/Q (28 Bits): Two 14-bit error correction blocks used to mathematically reconstruct damaged audio bits from scratch if the VHS tape has a dropout.
  • CRC-16 (16 Bits): A cryptographic checksum at the very end of the line used to aggressively verify if the line was corrupted by video noise.

1. Prerequisites & Installation

Install Java Runtime Environment (JRE)

This software is packaged as a standalone Java Archive (.jar). To run it, you must have Java 8 or newer installed on your computer.

  1. Visit the official Java Downloads Page.
  2. Download the Windows Offline (64-bit) installer.
  3. Run the installer and follow the standard on-screen instructions.
Install FFmpeg (Optional, Highly Recommended)

The application can natively encode silent MP4 black-and-white data streams entirely by itself using its built-in Java codecs. However, to embed a standard synchronous audio track into your final video file, you must use FFmpeg.

Option A: Using Winget (Easiest for Windows 10/11)

  1. Open Command Prompt or PowerShell and type: winget install ffmpeg
  2. Wait for the installation to complete. Winget will automatically configure your PATH variables.
  3. Restart your computer or terminal to ensure the command is recognized.

Option B: Manual Installation

  1. Download the FFmpeg Executable for Windows.
  2. Extract the downloaded archive and add the bin folder to your system's Environment Variables (PATH).
  3. Verify the installation by opening Command Prompt and typing ffmpeg -version.
Locate the Software

Once Java is installed, locate your compiled executable: PCMF1Simulator.jar.

Because the app writes video files directly to disk, it is recommended to place the `.jar` file in a dedicated folder (e.g. your Desktop or Documents) rather than deeply buried in system directories.

2. Running the Simulator

Option A: Double Click

If Java is correctly associated with `.jar` files in Windows, you can simply Double-Click PCMF1Simulator.jar to launch the graphical user interface.

Option B: Command Line (For Debugging)

If nothing happens when you double-click, or if you wish to see diagnostic output in a terminal, open Command Prompt (cmd) or PowerShell, navigate to the folder containing the jar, and run:

java -jar PCMF1Simulator.jar

3. Usage Instructions

A. Encoding Audio to Video (For VHS Recording)
  1. Open the application.
  2. Choose your preferred Encoder algorithm:
    • JCodec: Uses the pure Java internal engine. Generates a video-only STC-007 compliant MP4.
    • FFmpeg: (Requires FFmpeg system installation). Rapidly generates the video matrix and multiplexes it alongside a standard, listenable audio track for perfect media player pacing. Ensures universal audio format decoding.
  3. Ensure the Format dropdown matches your geographical standard (NTSC is standard for USA/Japan, PAL is standard for Europe). NTSC will process 44,056Hz PCM streams, while PAL processes standard 44,100Hz.
  4. Click Encode Audio.
  5. Select an audio file (.WAV, .FLAC, .MP3, .M4A). If using FFmpeg, compressed files are natively unpacked into raw 16-bit PCM automatically.
  6. The app will automatically begin packing the audio into a 720x480 interlaced video matrix. You will see the video preview generating in real-time on the main panel alongside amplitude readings on the retro VU meters.
  7. When complete, a mathematical match .mp4 will be saved in the same folder as your input file.
B. Decoding Video to Audio (Realtime Playback)
  1. Click Decode Audio.
  2. Select a valid STC-007 compliant MP4 video payload.
  3. The application will play the audio out of your computer speakers in real-time, syncing the hardware clocks directly to the NTSC framerate.
  4. The GUI will actively track corrected P/Q Parity dropout errors simulating true hardware recovery.
C. Quick Testing

Clicking Quick Test (440Hz) immediately fires a generated test sine wave through the encoder to verify your hardware limits and visually check UI performance without needing to hunt down a WAV file.

4. Hardware VHS Recording

Attention Video Enthusiasts: You cannot use cheap scaling "HDMI-to-RCA" converter boxes to print this MP4 to tape.

The output MP4 is a 720x480 NTSC or PAL Interlaced Signal. It relies on pure black-and-white binary edge retention. Modern composite scalers "smear" the picture to make it look smooth, which destroys the data.

To record the MP4 to tape successfully, you must output a raw unscaled 480i 15kHz signal:

  • Raspberry Pi: Play the MP4 through a Pi's native 3.5mm A/V composite combo jack.
